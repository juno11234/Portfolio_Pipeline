개요

4인 팀 프로젝트 담당 · 21 모듈 101 스크립트 Unity · C# · Rider 로그라이크 턴제 RPG

절차적으로 생성되는 노드 맵을 탐사하며 이벤트와 파티 턴제 전투를 진행하는 Slay the Spire식 로그라이크 RPG입니다. 저는 4인 팀에서 맵 생성 · 씬 흐름 · 전투 루프 · 연출 시스템 21개 모듈(101 스크립트)을 담당했습니다.

목표 ①구조 개선
이전 팀 프로젝트는 높은 결합도와 싱글톤 남용으로 실행 순서에 의존했습니다. 인터페이스 + DI(VContainer)로 의존성을 뒤집어 순서 의존을 제거합니다.
목표 ②프로세스 검증
AI 코딩 에이전트를 Plan · Build · Review 사이클로 도입해 실효성을 확인합니다. 리뷰 단계에서 결함을 사전에 걸러내는 것이 검증의 핵심입니다.
▶ 10초 요약 — 이 문서가 증명하는 것
DI 3계층 + 목적별 정적 채널로 씬 경계를 잇고, 단일 시드 결정론 맵 생성·다층 턴 루프·비파괴 Timeline 연출을 인터페이스 기반으로 구현했습니다. AI를 Plan·Build·Review 에이전트로 나눠 정적 분석으로 결함 3건을 리뷰 단계에서 제거했습니다.

내비게이션 노드 상태 전이 — 한 방향 경로

선택하지 않은 형제 경로는 영구 잠금됩니다. Slay the Spire식 “한 방향 경로”를 MoveToNode 하나로 구현한 상태 머신입니다.

Locked→ Available→ Current→ Visited 형제 경로 → Locked 영구
MapNavigator.cs · MoveToNode()
public bool MoveToNode(int nodeId)
{
    if (_mapData == null) return false;
    if (_mapData.NodeById.ContainsKey(nodeId) == false) return false;

    MapNode target = _mapData.NodeById[nodeId];
    // Available 상태가 아니면 이동 불가
    if (target.State != NodeState.Available) return false;

    // 이전 노드 정리: Visited + 선택하지 않은 다음 노드 잠금
    if (CurrentNode != null)
    {
        CurrentNode.State = NodeState.Visited;
        // Slay the Spire 방식: 다른 경로는 영구 잠금
        foreach (int nextId in CurrentNode.NextNodeIds)
            _mapData.NodeById[nextId].State = NodeState.Locked;
    }

    target.State = NodeState.Current;      // 새 노드를 Current로
    CurrentNode = target;
    UnlockNextNodes(target);                // 다음 층 Available
    OnNodeEntered?.Invoke(target);
    return true;
}
